UI panel which displays current artist's country on the world map
and lets you generate AutoPlaylists based on selection and locale
tag saving when integrated along WilB's Biography Script.
UI is fully configurable: colors, sizes, headers, flags (like the
ones found at Georgia Theme), etc.

UI usage:
---------
	• L. Click to show point menu for playlist creation or
	advanced point statistics.

	• Right panel buttons are only highlighted when mouse is over,
	otherwise they are partially hidden in the background.

	• Scroll with left/right buttons, horizontal mouse wheel or
	L. Click mouse dragging.

	• Ctrl + Alt + Mouse Wheel to resize UI elements under mouse.

	• Statistics via tooltip over points.

	• Zoom with mouse wheel and right panel button.

	• All tooltips show the available shortcuts.

	• Background is fully configurable.

	• Multiple sorting and filtering display options.

Other UI usage:
---------------
	• Ctrl + Alt + Mouse Wheel to resize UI elements under mouse.

	• Shift + Win + R. Click to open SMP/JSplitter panel menu (works
	globally on any script and panel, at any position).

	• Ctrl + Win + R. Click to open script panel menu.

Tags:
-----
	• Locale tag from currently playing/focused track(s).

	• May be a tag name or a TF expression.

	• May contain ISO values (ARG) or full names (Argentina).

	• Non-case sensitive.

	• Fetched any of these sources (by order):
		◦ File tags.
		◦ JSON data file (if available).
		◦ WilB's Biography Script (if enabled).

Biography:
----------
If tags are not found at file/JSON, they can be written they can be written
using the info shared by Biography Script for later offline usage.

Note that once the JSON file contains some artist's data, Internet
connection -and thus Biography Script too- is no longer needed... acting
as a local database for the tags related to those artists. An example
database is provided for that purpose, you may use it as an starting point
(look at 'presets' folder).

Map:
-----
Requires a world map using Mercator projection to work.

4 presets are given for convenience, but custom images are also allowed.

Note different maps may require different coordinates factors (since
height/width proportions may change). An example of use may be found
switching from 'Full' map to 'No Antarctica' Map, where the map is simply
cropped and the coordinates adjusted with an Y factor.

Library Mode:
-------------
All the countries from the entire library artists database are shown.
Clicking on a point creates playlists from that country (+genre, styles,
etc. according to keys pressed).

Tooltip shows all artist on library from that country, therefore being a
way to see the current statistics for every country.

Note this is not equivalent to selecting the entire library and using the
'Follow selection' mode for two reasons:
	• There is an configurable limit on point painting for selections
	(currently 5). This is done to reducecalculation time (it takes
	some time to retrieve tags and adjust layers).

	• Library mode is not calculated on real time, but using a cache.
	i.e. it's not really retrieving the tags from all tracks but just
	using their last known value (faster). May be updated on demand
	('Database\Update library database').

Statistics Mode (chart):
------------------------
Full library stats are displayed in a chart: both data and display settings
are configurable using the buttons. Clicking on any point allows to create
playlists or show further stats via menus.

Statistics Mode (gradient map):
-------------------------------
Library stats are displayed over the map, using a gradient map with country
layers whose color changes according to the number of artists from that
country. It's pretty memory/resource intensive.

Other tools:
-------------
A set of tools is available to easily merge different json databases, write
tags to json or back from json to files, find items in the library without
the locale tags or not present on database, etc.

RAM usage:
----------
SMP/JSplitter may throw "Out of memory" errors in some settings or when
using too many panels at the same time. To minimize RAM usage, there are
multiple memory modes available (with explanations about the changes).
